# Бизнес-логика приложения «Правовой маршрутизатор»

---

## Цель приложения

Помочь сотруднику без знания внутренних регламентов за несколько шагов определить:
- куда направлять обращение;
- что именно указать в обращении;
- какие документы приложить;
- в какой срок ожидать ответ.

---

## Общая структура приложения

```
Главный экран
├── Договорная работа
│   ├── Новый договор
│   └── Дополнительное соглашение (ДС)
├── Претензионная работа
│   ├── Претензия к контрагенту
│   └── Ответ на претензию контрагента
└── База знаний
```

---

## Системные элементы (присутствуют на всех экранах)

| Элемент | Описание |
|---|---|
| Индикатор категории (хлебные крошки) | Путь пользователя: `Договорная работа → Новый договор` |
| Кнопка «Назад» | Возврат на предыдущий экран без потери ответов |
| Кнопка «Начать заново» | Сброс сессии, возврат на главный экран |

---

## Стандарт экрана результата (единый для всех маршрутов)

На финальном экране каждого маршрута пользователь получает **два документа**:

### Документ 1 — Обращение в юридическую службу (PDF)

Формируется **полностью автоматически** из:
- данных авторизации пользователя (ФИО, подразделение, контактные данные);
- ответов на вопросы маршрута.

Ручной ввод не требуется.

Структура (единая для всех маршрутов):

| Поле | Источник |
|---|---|
| Кому | Статично — администратор |
| От кого | Авторизация: ФИО |
| Подразделение | Авторизация: подразделение |
| Контактные данные | Авторизация: email / телефон |
| Тип запроса | Автозаполнение по маршруту |
| Контрагент | Ответ пользователя |
| Предмет договора / ДС | Ответ пользователя |
| Реквизиты договора | Ответ пользователя (для претензионной работы — обязательно) |
| Отчётный период | Ответ пользователя (для претензионной работы — для идентификации) |
| Основание обращения | Автозаполнение по маршруту |
| Срок согласования юристом | Статично — администратор |

> **Примечание для разработки:** в перспективе возможна интеграция с 1С:ДО — автозаполнение полей карточки документа из Документа 1 приложения.

### Документ 2 — Инструкция-чеклист для сотрудника (PDF)

Содержит:
1. Что собрать и приложить
2. Что должно быть указано в проекте документа
3. Как инициировать согласование в 1С:ДО
4. Предельный срок (для ответа на претензию — расчётный)

---

## Инструкция по работе в 1С:ДО (включается в Документ 2 всех маршрутов)

> **Как инициировать согласование в 1С:ДО:**
> 1. Создайте карточку документа:
>    - договорная работа → тип **«Внутренний документ» → вид «Договор (расходный)»**
>    - претензионная работа → тип **«Исходящее письмо»**
> 2. Заполните обязательные поля карточки (контрагент, предмет, реквизиты договора).
> 3. Прикрепите все документы согласно чеклисту.
> 4. В маршрут согласования добавьте: своего руководителя, юриста.
> 5. Запустите процесс согласования.
